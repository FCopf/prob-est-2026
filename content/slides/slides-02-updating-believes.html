Calibrando a Razão $\Lambda$

O que esperamos quando H₁ é verdadeira?

Uma razão $\Lambda$ é grande ou pequena?

A razão de verossimilhanças mede a compatibilidade relativa de dois modelos com os dados. Para interpretar o valor obtido, precisamos de um ponto de referência: o que esperamos obter apenas pelo acaso quando H₁ é o mecanismo verdadeiro?

Se gerarmos muitos conjuntos de dados hipotéticos sob H₁, veremos qual distribuição de $\log\Lambda$ surge naturalmente pelo acaso. O valor observado é compatível com essa distribuição?

⏱️ 2 min. Conceito de distribuição de referência. Explicar que vamos gerar 1000 experimentos hipotéticos sob H₁.

Simulando 1000 experimentos sob H₁


# Parâmetros da simulação
n_sim <- 1000
n_obs <- length(freq_mental)   # mesmo tamanho do experimento real

# Simular log(Lambda) para cada experimento hipotético sob M1
log_razoes_sim <- replicate(n_sim, {
  amostra  <- sample(1:6, size = n_obs, replace = TRUE, prob = p_h1)
  freq_sim <- table(factor(amostra, levels = 1:6))
  ll_h1    <- sum(freq_sim * log(p_h1))
  ll_h2    <- sum(freq_sim * log(p_h2))
  ll_h2 - ll_h1
})

Verificação Preditiva

Os dados parecem ter sido gerados por H₂?

Uma segunda pergunta sobre os modelos

Mostramos que H₂ é mais verossímil que H₁. Mas isso ainda não responde: H₂ descreve bem o processo gerador dos dados?

A verificação preditiva consiste em gerar dados simulados sob o modelo candidato e verificar se eles se assemelham visualmente aos dados reais. Se M₂ captura algo real, suas amostras devem se parecer com o padrão observado.

Conceito de verificação preditiva. É um diagnóstico qualitativo do modelo. Complementa — e não substitui — a razão de verossimilhanças.

Atualização Bayesiana

Incorporando o Conhecimento Prévio

Da Votação à Priori

Na Aula 01, antes de ver os dados, vocês votaram:
o dado mental seguirá H₁ ou H₂?

Esse voto expressa a crença coletiva da turma em cada hipótese antes de observar qualquer resultado. Em inferência bayesiana, essa crença prévia recebe o nome de distribuição a priori.

A priori só faz sentido quando há pelo menos duas hipóteses para comparar.

Apontar para os números anotados no quadro na Aula 01. A votação anterior É a priori — não é uma analogia.

Os três componentes da inferência bayesiana

Priori $P(H)$

Crença na hipótese antes de observar os dados.
Vem de estudos anteriores, teoria ou consenso.

Verossimilhança $\mathcal{L}(H \mid \text{dados})$

Compatibilidade da hipótese com os dados observados.
Determinada pelos dados e pelo modelo.

Posteriori $P(H \mid \text{dados})$

Crença depois de incorporar a evidência.
Resultado da atualização bayesiana.

$$P(H \mid \text{dados}) \propto P(H) \times \mathcal{L}(H \mid \text{dados})$$
Apresentar os três componentes antes da fórmula formal. A proporção é a versão intuitiva da regra de Bayes.

A Regra de Bayes

Para duas hipóteses $H_1$ e $H_2$ com probabilidades a priori $P(H_1)$ e $P(H_2) = 1 - P(H_1)$:

$$P(H_j \mid \text{dados}) = \frac{P(H_j) \cdot \mathcal{L}(H_j \mid \text{dados})}{\displaystyle\sum_{i} P(H_i) \cdot \mathcal{L}(H_i \mid \text{dados})}$$

O denominador é uma constante de normalização que garante $P(H_1 \mid \text{dados}) + P(H_2 \mid \text{dados}) = 1$.
O que importa é a relação entre os numeradores.

Mostrar a fórmula completa com o denominador. Enfatizar que a posteriori é proporcional ao produto priori × verossimilhança. O denominador apenas normaliza para as probabilidades somarem 1.

Codificando a priori da turma


# Priori da turma — substituir pelos valores reais da votação da Aula 01
votos_h1 <- 32   # numero de votos para H1
votos_h2 <- 18   # numero de votos para H2
n_turma  <- votos_h1 + votos_h2

prior <- c(H1 = votos_h1 / n_turma,
           H2 = votos_h2 / n_turma)
cat("Priori da turma:\n")
print(round(prior, 3))
No exemplo acima: $P(H_1) = 0{,}64$, $P(H_2) = 0{,}36$
Substituir pelos valores REAIS da votação da Aula 01. Mostrar que a priori soma 1.

Calculando a posteriori


# Verossimilhanças (na escala original, não logarítmica)
lik <- c(H1 = exp(loglik_h1_mental),
         H2 = exp(loglik_h2_mental))

# Posteriori: priori × verossimilhança, normalizada
post <- prior * lik
post <- post / sum(post)

cat("Priori:    H1 =", round(prior[1], 3), "| H2 =", round(prior[2], 3), "\n")
cat("Posteriori: H1 =", round(post[1],  3), "| H2 =", round(post[2],  3), "\n")
Multiplicar priori × verossimilhança elemento a elemento. Dividir pelo total para normalizar. Mostrar que o resultado muda em relação à priori.

O Papel da Priori

Diferentes crenças iniciais, diferentes conclusões?

Três cenários de priori

Com os mesmos dados e a mesma verossimilhança, como diferentes prioris alteram a posteriori? 


# Verossimilhanças (obtidas dos dados)
lik <- c(H1 = exp(loglik_h1_mental), H2 = exp(loglik_h2_mental))

# Cenário 1: priori neutra — nenhuma hipótese é preferida a priori
prior_neutra  <- c(H1 = 0.5,  H2 = 0.5)

# Cenário 2: priori favorece fortemente H1
prior_forte_h1 <- c(H1 = 0.9, H2 = 0.1)

# Cenário 3: priori favorece fortemente H2
prior_forte_h2 <- c(H1 = 0.1, H2 = 0.9)

# Calcular as posterioris para cada cenário
post_neutra   <- (prior_neutra   * lik) / sum(prior_neutra   * lik)
post_forte_h1 <- (prior_forte_h1 * lik) / sum(prior_forte_h1 * lik)
post_forte_h2 <- (prior_forte_h2 * lik) / sum(prior_forte_h2 * lik)

round(rbind(post_neutra, post_forte_h1, post_forte_h2), 3)
⏱️ 3 min. Rodar ao vivo e discutir os três resultados.

O que muda com cada priori?

Priori neutra
$P(H_1) = P(H_2) = 0{,}5$

A posteriori reflete diretamente a razão das verossimilhanças.
H₂ recebe probabilidade maior porque os dados a favorecem.

Priori forte em H₁
$P(H_1) = 0{,}9$

A crença inicial forte em H₁ resiste à evidência.
A posteriori pode ainda favorecer H₁ mesmo que os dados favoreçam H₂.

Priori forte em H₂
$P(H_2) = 0{,}9$

Priori e evidência apontam na mesma direção.
H₂ recebe apoio ainda maior na posteriori.

Quando a evidência é muito forte, a priori tem pouca influência. Quando a evidência é fraca ou ambígua, a priori decide o resultado.
Discutir cada cenário com os valores calculados no R. Propriedade importante: com evidência suficiente, pesquisadores com prioris diferentes convergem para a mesma conclusão.

Priori neutra: uma propriedade especial

Quando as duas hipóteses recebem probabilidades a priori iguais ($P(H_1) = P(H_2) = 0{,}5$), a razão entre as probabilidades a posteriori coincide exatamente com a razão de verossimilhanças:

$$\frac{P(H_2 \mid \text{dados})}{P(H_1 \mid \text{dados})} = \frac{P(H_2)}{P(H_1)} \times \frac{\mathcal{L}(H_2)}{\mathcal{L}(H_1)} = 1 \times \Lambda = \Lambda$$
Com priori neutra, a atualização bayesiana simplesmente normaliza as verossimilhanças para que se tornem probabilidades que somam 1.
Essa propriedade conecta a razão de verossimilhanças (Aula 01) com a atualização bayesiana (hoje). Com priori neutra, $\Lambda$ já determina a posteriori.

O Ciclo Completo

da Inferência Científica

O caminho da inferência

1. Formular hipóteses
Especificar pelo menos duas hipóteses concorrentes antes de coletar dados.
2. Traduzir em modelos probabilísticos
Para cada hipótese, definir as probabilidades de cada resultado possível.
3. Observar os dados
Coletar as contagens reais e organizá-las por categoria.
4. Calcular a verossimilhança
Para cada modelo: $\sum n_k \cdot \log(p_k)$.
5. Comparar as hipóteses
Obter $\Lambda = \mathcal{L}(H_2) / \mathcal{L}(H_1)$.
6. Calibrar com simulação
Gerar dados sob H₁ para avaliar se $\Lambda$ é surpreendente ou esperado.
7. Verificação preditiva
Simular dados sob H₂ e verificar se reproduzem o padrão observado.
8. Atualizar as crenças
Combinar priori e verossimilhança para obter a posteriori via regra de Bayes.
Percorrer cada etapa e conectar com o que fizemos nas duas aulas. As etapas 1–5 foram cobertas na Aula 01. As etapas 6–8 foram cobertas hoje.

🔄 O Ciclo de Mecanismo Bayesiano

Crença Inicial × Evidência = Crença Atualizada

A priori distribui a crença entre as hipóteses antes dos dados.
A verossimilhança redistribui essa crença à luz da evidência.
A posteriori é a crença atualizada, pronta para o próximo ciclo.

A posteriori de hoje é a priori de amanhã

Esse é o poder da inferência bayesiana. O ciclo se repete cada vez que novos dados chegam. Com mais dados, a priori inicial perde importância e os dados assumem o controle.
