O modelo Normal da Aula 05 — revisão

Dataset: Alturas de adultos !Kung San. Dados: McElreath, 2020.
$$h_i \sim \text{Normal}(\mu, \sigma)$$ $$\mu \sim \text{Normal}(160, 20)$$ $$\sigma \sim \text{Exponencial}(0{,}1)$$
Limitação: $\mu$ é um escalar único. Dois adultos com pesos de 50 kg e 90 kg recebem a mesma média esperada de altura — o modelo ignora a associação peso-altura.
⏱️ 3 min. Revisitar o modelo da Aula 05 para estabelecer o ponto de partida. Enfatizar que o símbolo ~ indica relação estocástica: h_i é amostrado da distribuição Normal. O problema: a média mu é constante para todas as observações, independentemente do peso. Perguntar: "Se dois adultos têm pesos muito diferentes, faz sentido que o modelo preveja a mesma altura média para ambos?"

A Substituição Central

De $\mu$ para $\mu_i$ — o preditor linear

Esta parte apresenta a mudança formal que transforma o modelo Normal em modelo de regressão linear. É a parte mais importante da aula do ponto de vista conceitual. Tempo estimado: 10 min.

A substituição central

Modelo Normal (Aula 05)

$$h_i \sim \text{Normal}(\mu, \sigma)$$ $$\mu \sim \text{Normal}(160, 20)$$

$\mu$ é um escalar — igual para todos

→

Modelo Linear (Aula 08)

$$h_i \sim \text{Normal}(\mu_i, \sigma)$$ $$\mu_i = \beta_0 + \beta_1(x_i - \bar{x})$$

$\mu_i$ varia entre observações

O subscrito $i$ em $\mu_i$ indica que a média esperada é calculada individualmente para cada observação, a partir do peso daquele indivíduo.
⏱️ 3 min. A substituição central é o coração desta aula. Enfatizar que a diferença entre os dois modelos é apenas o subscrito i em mu. Essa pequena mudança na notação tem consequências profundas: agora a média esperada varia entre observações. Chamar atenção para o símbolo = na segunda equação: o preditor linear é determinístico, não estocástico.

O preditor linear

$$\mu_i = \beta_0 + \beta_1(x_i - \bar{x})$$

O símbolo $=$ (e não $\sim$) indica que o preditor linear é uma relação determinística. Dados $\beta_0$, $\beta_1$ e o peso $x_i$, o valor de $\mu_i$ é completamente calculado.


É chamado linear porque é uma função linear dos parâmetros $\beta_0$ e $\beta_1$, independentemente da forma matemática da covariável.

⏱️ 2 min. Reforçar a distinção entre relação estocástica (til) e relação determinística (igual). No modelo, apenas h_i tem distribuição de probabilidade: é amostrado da Normal. O valor de mu_i é computado, não amostrado. Essa distinção é a principal dificuldade conceitual desta aula.

Três componentes do modelo linear bayesiano

Componente Expressão Papel
Distribuição $h_i \sim \text{Normal}(\mu_i, \sigma)$ Incerteza estocástica sobre cada observação
Preditor linear $\mu_i = \beta_0 + \beta_1(x_i - \bar{x})$ Média esperada como função determinística da covariável
Distribuições a priori $\beta_0 \sim \ldots,\;\beta_1 \sim \ldots,\;\sigma \sim \ldots$ Crenças sobre os parâmetros antes dos dados
Os únicos parâmetros com distribuições a posteriori são $\beta_0$, $\beta_1$ e $\sigma$. O valor $\mu_i$ é calculado a partir deles, não é estimado separadamente.
⏱️ 2 min. Apresentar o quadro dos três componentes. Enfatizar que o modelo tem apenas três parâmetros livres, não 352 (um mu_i por observação). Perguntar: "Quantos parâmetros têm distribuições a posteriori neste modelo?" Resposta esperada: três — beta_0, beta_1, sigma.

$\mu_i$ não é um parâmetro livre

Em um conjunto com 352 adultos, $\mu_i$ assumiria 352 valores distintos.

Esses 352 valores não são 352 parâmetros a estimar.

Cada valor $\mu_i$ é calculado a partir de apenas três parâmetros:

$$\mu_i = \beta_0 + \beta_1 \cdot (x_i - \bar{x})$$

Apenas $\beta_0$, $\beta_1$ e $\sigma$ têm distribuições a posteriori.

⏱️ 2 min. Este é o ponto de atenção mais frequente. Estudantes leem mu_i com subscrito i e concluem que há 352 parâmetros a estimar. Reforçar: mu_i é uma quantidade calculada, derivada dos três parâmetros reais. Analogia: a temperatura prevista para cada hora do dia é calculada a partir de um modelo meteorológico com poucos parâmetros, não de 24 parâmetros independentes.

Enquete 1

No modelo $h_i \sim \text{Normal}(\mu_i, \sigma)$ com $\mu_i = \beta_0 + \beta_1 \cdot (x_i - \bar{x})$, qual afirmação está correta?

  • (A) $\mu_i$ é um parâmetro estocástico com distribuição a posteriori própria.
  • (B) $\mu_i$ é calculado deterministicamente a partir de $\beta_0$, $\beta_1$ e $x_i$.
  • (C) $\sigma$ captura a incerteza sobre $\beta_0$ e $\beta_1$.
  • (D) $\beta_0$ é o valor esperado de $h_i$ quando $\beta_1 = 0$.
⏱️ 2 min para a enquete. Resposta correta: (B). Discutir a distinção entre o símbolo til (relação estocástica) e o símbolo igual (relação determinística). A alternativa (D) é uma armadilha: quando beta_1 = 0, mu_i = beta_0 para todos os indivíduos, mas essa não é a interpretação canônica do intercepto no modelo com centralização. Voltar ao quadro dos três componentes para reforçar.

Centralização da Covariável

Interpretando $\beta_0$ de forma direta

Esta parte apresenta a centralização da covariável e a interpretação dos três parâmetros. O objetivo é que o estudante compreenda o que cada parâmetro representa em termos observáveis. Tempo estimado: 8 min.

O problema sem centralização

$$\mu_i = \beta_0 + \beta_1 x_i$$

Nessa forma, $\beta_0$ é a altura esperada quando $x_i = 0$.

Para adultos humanos, peso igual a zero kg é biologicamente impossível.

O intercepto perde a interpretação direta.

⏱️ 2 min. Mostrar o problema prático do intercepto sem centralização. Um peso de zero quilogramas é impossível para um adulto vivo. A distribuição a priori para beta_0 teria que cobrir uma faixa biologicamente implausível.

A solução pela centralização

Definir a covariável centralizada: $\tilde{x}_i = x_i - \bar{x}$
$$\mu_i = \beta_0 + \beta_1 \tilde{x}_i$$
Sem centralização
$\beta_0$ = altura esperada quando $x_i = 0$
Biologicamente impossível
Com centralização
$\beta_0$ = altura esperada quando $\tilde{x}_i = 0$
= peso médio da amostra
⏱️ 2 min. Explicar a transformação de centralização. O valor zero na escala centralizada corresponde ao peso médio da amostra. Enfatizar: a centralização muda o ponto de referência do eixo x. A relação entre peso e altura permanece idêntica — apenas o ponto de ancoragem muda.

O que muda e o que permanece

Parâmetro Com centralização Centralização altera?
$\beta_0$ (intercepto) Altura esperada no peso médio Sim — valor muda
$\beta_1$ (coeficiente de regressão) Mudança em altura por kg adicional Não — idêntico
$\sigma$ (desvio padrão residual) Dispersão em torno da reta ajustada Não — idêntico
A centralização move o ponto de referência do eixo $x$, sem alterar a inclinação nem a dispersão residual.
⏱️ 2 min. Tabela de o que muda e o que permanece. Este é o ponto da pergunta diagnóstica do roteiro. Perguntar: "Se centralizarmos o peso antes de ajustar o modelo, quais parâmetros mudam?" Resposta: apenas beta_0 muda. beta_1 e sigma permanecem idênticos. Analogia: trocar o zero da escala de temperatura (Celsius para Fahrenheit muda o ponto de referência, mas não as diferenças).

Interpretação dos três parâmetros

$\beta_0$ (intercepto) altura esperada de um adulto com peso igual à média da amostra.
$\beta_1$ (coeficiente de regressão) mudança esperada na altura a cada aumento de 1 kg no peso centralizado.
$\beta_1 > 0$ indica que indivíduos mais pesados são, em média, mais altos.
$\sigma$ (desvio padrão residual) dispersão das alturas em torno da reta ajustada. Captura a variabilidade individual não explicada pelo preditor linear.
⏱️ 2 min. Concluir a parte de centralização com a interpretação dos três parâmetros. Esta interpretação é diretamente aplicável aos resultados que os estudantes verão no laboratório. Pedir que associem cada parâmetro a uma pergunta biológica concreta.

Distribuições A Priori

Conhecimento de domínio antes dos dados

Esta parte apresenta as distribuições a priori para os três parâmetros do modelo linear. O ponto mais importante é a justificativa para a escolha da Distribuição Lognormal para beta_1. Tempo estimado: 10 min.

Distribuição a priori para $\beta_0$

$$\beta_0 \sim \text{Normal}(160, 20)$$

Com o peso centralizado, $\beta_0$ é a altura esperada de um adulto com peso médio.

A distribuição Normal centrada em 160 cm concentra 95% da probabilidade entre 120 e 200 cm, intervalo mais amplo do que o plausível para adultos.

É uma distribuição a priori fracamente informativa: cobre a faixa biologicamente razoável sem restringir demais.

⏱️ 2 min. A distribuição a priori de beta_0 é análoga à de mu na Aula 05. Enfatizar que "fracamente informativa" significa que ainda permite valores biologicamente impossíveis, mas com baixa probabilidade. Mostrar que 120-200 cm é deliberadamente mais amplo que o razoável para adultos (~150-200 cm).

Distribuição a priori para $\beta_1$ — por que Lognormal?

Normal(0, 10) — problemática
Atribui probabilidade substancial a valores negativos.
Implica que pessoas mais pesadas podem ser mais baixas — biologicamente implausível.
Lognormal(0, 1) — adequada
Definida apenas para valores positivos.
Mediana: $e^0 = 1$ cm/kg.
95% da distribuição: entre 0,1 e 7,4 cm/kg.
$$\beta_1 \sim \text{Lognormal}(0, 1)$$
$X \sim \text{Lognormal}(\mu, \sigma) \iff \log(X) \sim \text{Normal}(\mu, \sigma)$. A Distribuição Lognormal é definida apenas para valores positivos ($X > 0$).
⏱️ 3 min. Este é o ponto mais importante desta parte. A escolha de Lognormal é um ato epistêmico — reflete o conhecimento de que a relação altura-peso em adultos é positiva. Se o conhecimento de domínio não informasse o sinal do coeficiente, a Normal seria mais adequada. Isso ficará claro no laboratório: para riqueza de espécies em função do nível de maré, o efeito esperado é negativo, portanto a Lognormal seria inadequada.

Distribuição a priori para $\sigma$

$$\sigma \sim \text{Exponencial}(1)$$

O desvio padrão residual deve ser positivo. A Distribuição Exponencial fornece essa restrição de forma natural.

Para $\sigma \sim \text{Exponencial}(1)$, a mediana é aproximadamente 0,69 cm. Cerca de 95% da probabilidade está entre 0,03 e 3,7 cm, definindo um intervalo fracamente informativo.

Resumo das distribuições a priori:
$\beta_0 \sim \text{Normal}(160, 20)$    $\beta_1 \sim \text{Lognormal}(0, 1)$    $\sigma \sim \text{Exponencial}(1)$
⏱️ 2 min. Encerrar a apresentação das distribuições a priori com o resumo. Enfatizar que cada distribuição a priori reflete o conhecimento de domínio disponível sobre o parâmetro correspondente.

Enquete 2

Por que $\beta_1 \sim \text{Lognormal}(0, 1)$ é preferível a $\beta_1 \sim \text{Normal}(0, 10)$ neste modelo?

  • (A) Porque a Lognormal é matematicamente mais fácil de especificar no brms.
  • (B) Porque a Lognormal é sempre mais estreita e reduz os intervalos de credibilidade.
  • (C) Porque a Lognormal atribui toda a probabilidade a valores positivos, refletindo que pessoas mais pesadas tendem a ser mais altas.
  • (D) Porque a Normal(0, 10) não é compatível com os dados observados.
⏱️ 2 min para a enquete. Resposta correta: (C). Enfatizar: a escolha de distribuição a priori é um ato epistêmico — ela reflete o que se sabe antes de ver os dados. Quando o conhecimento de domínio indica que o coeficiente deve ser positivo, a Lognormal é a escolha epistemicamente justificada. A alternativa (D) é incorreta: a distribuição a priori é especificada antes de ver os dados e não depende da compatibilidade com os dados.

Checagem A Priori e Ajuste do Modelo

Feixe de retas, distribuição a posteriori e diagnóstico

Esta parte apresenta o fluxo completo de ajuste bayesiano do modelo linear. Da checagem preditiva a priori até a distribuição a posteriori e diagnóstico de convergência. Tempo estimado: 12 min (inclui transição para o laboratório).

Checagem preditiva a priori

Antes de ver os dados, as distribuições a priori definem um espaço de retas plausíveis.

Amostrar pares $(\beta_0^{(s)}, \beta_1^{(s)})$ das distribuições a priori e traçar as retas correspondentes sobre o diagrama de dispersão.

Aula 05 — modelo Normal
Checagem produzia uma distribuição de alturas simuladas
Aula 08 — modelo linear
Checagem produz um feixe de retas simuladas
⏱️ 2 min. Apresentar a extensão da checagem preditiva a priori para o modelo linear. No modelo Normal, a checagem a priori gerava uma distribuição de alturas. No modelo linear, ela gera um conjunto de retas — uma para cada par (beta_0, beta_1) amostrado das distribuições a priori. O feixe resultante deve cobrir alturas plausíveis e ter inclinações positivas moderadas.

Código — checagem preditiva a priori

library(brms)
library(posterior)

prioris <- c(
  prior(normal(160, 20),   class = Intercept),
  prior(lognormal(0, 1),   class = b, coef = weight_c),
  prior(exponential(1),    class = sigma)
)

# Ajuste usando apenas as distribuições a priori
fit_prior <- brm(
  formula = height ~ weight_c,
  family  = gaussian(),
  prior   = prioris,
  data    = adultos,
  sample_prior = "only",
  chains = 4, iter = 2000, warmup = 1000, refresh = 0
)

# Feixe de retas simuladas
amostras_prior <- as_draws_df(fit_prior)
ggplot(adultos, aes(x = weight_c, y = height)) +
  geom_point(alpha = 0.2, color = "#1a9988") +
  geom_abline(
    data = amostras_prior[1:50, ],
    aes(intercept = b_Intercept, slope = b_weight_c),
    alpha = 0.2, color = "#2c5f7c"
  ) + theme_bw()
O argumento sample_prior = "only" ajusta o modelo sem observar os dados — as amostras vêm exclusivamente das distribuições a priori.
⏱️ 2 min. Apresentar o código da checagem preditiva a priori. Chamar atenção para sample_prior = "only": os dados estão presentes no data= mas não são usados na estimação. Alertar sobre a compilação do Stan: na primeira chamada de brm() na sessão, o código Stan será compilado em C++ (30-90 segundos). Isso é normal e esperado. O feixe de retas deve cobrir alturas biologicamente plausíveis e ter inclinações positivas moderadas.

Ajuste do modelo com os dados

# Remover sample_prior = "only" para ajustar com os dados observados
fit <- brm(
  formula = height ~ weight_c,
  family  = gaussian(),
  prior   = prioris,
  data    = adultos,
  chains  = 4, iter = 2000, warmup = 1000, refresh = 0
)

# Resumo da distribuição a posteriori — intervalo de credibilidade de 89%
amostras <- as_draws_df(fit)
summarise_draws(
  amostras,
  mean, sd,
  ~quantile(.x, c(0.055, 0.945))
)
A única mudança em relação à checagem a priori é a remoção de sample_prior = "only". A estrutura do modelo, as distribuições a priori e as cadeias permanecem idênticas.
⏱️ 2 min. Destacar que a mudança de código é mínima — remove-se apenas um argumento. Isso reforça que a checagem preditiva a priori e o ajuste completo são etapas do mesmo fluxo. O resumo da distribuição a posteriori com intervalo de 89% é o padrão utilizado no curso.

Diagnóstico de convergência

summary(fit)

Antes de interpretar qualquer resultado, verificar dois critérios:

  • R-hat ($\hat{R}$) $<$ 1,01 para todos os parâmetros. Valores maiores indicam que as cadeias não convergiram para a mesma distribuição.
  • Bulk ESS $>$ 400 para todos os parâmetros. Valores menores indicam alta correlação entre amostras e estimativas imprecisas.
⏱️ 2 min. O diagnóstico de convergência deve sempre preceder a interpretação substantiva dos parâmetros. O summary(fit) exibe rhat, ess_bulk e ess_tail para cada parâmetro. Enfatizar: resultados de um modelo que não convergiu não devem ser interpretados. Este procedimento é idêntico ao que os estudantes aprenderão no laboratório com os dados RIKZ.

Faixa de Credibilidade e Intervalo Preditivo

Duas formas distintas de incerteza

Esta parte apresenta a distinção entre a faixa de credibilidade da resposta média e o intervalo preditivo. É a distinção mais frequentemente confundida na prática — e também a mais importante para a interpretação do gráfico do laboratório. Tempo estimado: 8 min.

Duas formas de incerteza sobre a reta

Faixa de credibilidade (banda azul)

Incerteza sobre onde está a reta verdadeira.

Intervalo de credibilidade sobre $\mu_i = E[h \mid x_i]$.

Calculada com posterior_epred().

Intervalo preditivo (banda laranja)

Incerteza sobre uma nova observação individual.

Combina incerteza da reta com variabilidade individual ($\sigma$).

Calculada com posterior_predict().

O intervalo preditivo é sempre mais largo do que a faixa de credibilidade, porque inclui uma fonte adicional de incerteza ($\sigma$).
⏱️ 2 min. Apresentar a distinção fundamental entre as duas bandas. Analogia: a incerteza sobre a temperatura média amanhã (faixa de credibilidade) é menor do que a incerteza sobre a temperatura que um termômetro específico registrará naquela hora (intervalo preditivo). A faixa de credibilidade depende apenas da incerteza sobre beta_0 e beta_1. O intervalo preditivo inclui adicionalmente a variabilidade individual sigma.

Construindo as duas bandas — código

library(bayesplot)

# Sequência de pesos para predição
weight_seq <- seq(min(adultos$weight_c), max(adultos$weight_c), length.out = 100)
novos_dados <- data.frame(weight_c = weight_seq)

# Faixa de credibilidade da resposta média
epred  <- posterior_epred(fit, newdata = novos_dados)
# Intervalo preditivo para novas observações
pred   <- posterior_predict(fit, newdata = novos_dados)

resumo_epred <- data.frame(
  weight_c = weight_seq,
  media = apply(epred, 2, mean),
  q_inf = apply(epred, 2, quantile, 0.055),
  q_sup = apply(epred, 2, quantile, 0.945)
)
resumo_pred <- data.frame(
  weight_c = weight_seq,
  q_inf = apply(pred, 2, quantile, 0.055),
  q_sup = apply(pred, 2, quantile, 0.945)
)
⏱️ 2 min. Apresentar o código para calcular as duas bandas. Chamar atenção para a diferença entre posterior_epred (media esperada) e posterior_predict (nova observação). Os quantis 5,5% e 94,5% definem o intervalo de credibilidade de 89% — padrão do curso.

Visualização das duas bandas

ggplot(adultos, aes(x = weight_c, y = height)) +

  geom_ribbon(data = resumo_pred,

A maioria dos pontos observados deve estar dentro da banda laranja (intervalo preditivo). Pontos sistematicamente fora indicam que o modelo não capta adequadamente a distribuição dos dados.
⏱️ 2 min. Apresentar o gráfico resultante. A banda azul (faixa de credibilidade) é mais estreita e descreve onde está a reta. A banda laranja (intervalo preditivo) é mais larga e descreve onde se espera encontrar os dados. Os pontos observados devem, em sua maioria, cair dentro da banda laranja.

Checagem preditiva da posteriori

pp_check(fit, ndraws = 100)

A linha escura representa a distribuição observada de alturas.

As linhas coloridas representam distribuições simuladas a partir do modelo ajustado.

Sobreposição próxima indica que o modelo capta adequadamente a forma da distribuição dos dados.

⏱️ 2 min. Apresentar a checagem preditiva da posteriori com pp_check(). Reforçar a distinção entre checagem a priori (antes dos dados) e checagem da posteriori (após o ajuste).
