Do Dado ao Globo

Retomada da Aula 01 e apresentação do problema central

⏱️ 20 min. Esta parte cobre a abertura (5 min) e a Teoria 1 (15 min). Comece projetando a imagem de um globo terrestre. A pergunta-gatilho deve ser lançada antes de qualquer fórmula nova.

O que fizemos na Aula 01

Ponto de partida
  • Duas hipóteses: $H_0$ (dado justo, $p = 1/6$) e $H_1$ (dado viciado, $p = 4/14$)
  • Calculamos $\mathcal{L}(H \mid \text{dados})$ para cada hipótese
  • Combinamos priori e verossimilhança para obter a posteriori
A regra geral

$$P(H \mid \text{dados}) \propto \mathcal{L}(H \mid \text{dados}) \times P(H)$$

Priori $\times$ Verossimilhança $\to$ Posteriori

Pergunta-gatilho: E se o parâmetro não for nem $1/6$ nem $4/14$, e sim qualquer valor entre 0 e 1?
⏱️ 5 min. Projete um globo terrestre ao fundo. Retome os dois pesos de posteriori da Aula 01 rapidamente — não reconstrua o cálculo do zero. Pergunte: "Se eu adicionasse um terceiro modelo $H_2$, como ficaria o cálculo?" Deixe os alunos generalizar mentalmente. Esse é o ponto de partida para o grid.

O experimento do globo

Jogamos um globo para cima e registramos se o dedo pousa em água (W) ou em terra (L).

W   L   W   W   L   W   W   W   L

$k = 6$ águas, $n - k = 3$ terras, $n = 9$ lançamentos

O parâmetro de interesse

$p$ = proporção de água na superfície do globo, $p \in [0, 1]$

A pergunta

Como distribuir nossa incerteza sobre $p$ depois de observar os dados?

⏱️ 3 min. Se possível, use um globo inflável físico. Peça a um aluno para lançar e registrar o resultado. Anote W ou L no quadro. Repita até ter 9 lançamentos. Se não for possível, use os valores k=6, n=9 diretamente. Diga: "Não sabemos p. Poderia ser 0.10, poderia ser 0.92. Hoje vamos aprender a expressar essa incerteza."

De duas hipóteses a infinitas

Aula 01: dois pontos no espaço paramétrico

$H_0$: $p = 1/6$
$H_1$: $p = 4/14$

Hoje: um grid de $G$ pontos cobrindo $[0, 1]$

Grid (10 pontos) $p = 0.0$ $p = 0.1$ $p = 0.2$ $\ldots$ $p = 1.0$
Hipótese $H_1$ $H_2$ $H_3$ $\ldots$ $H_{10}$
Cada ponto do grid é uma hipótese separada — a mesma lógica da Aula 01, agora com 10 (ou 100, ou 1000) hipóteses simultâneas.
⏱️ 4 min. Este é o slide de conexão mais importante. Diga explicitamente: "Antes, H0 e H1 eram dois pontos. Agora temos 10 pontos — ou 1000, se quisermos mais precisão. A lógica é idêntica: calculamos a verossimilhança de cada hipótese, multiplicamos pela priori, normalizamos." Pergunte: "Antes de ver os dados, qual valor de p você acha que o globo tem?" Anote as respostas dos alunos.

O modelo binomial

Cada lançamento resulta em W ou L com probabilidade $p$. O modelo adequado é a distribuição binomial.

$$P(k \mid n, p) = \binom{n}{k} p^k (1-p)^{n-k}$$
$n$
número de tentativas
(fixo: sabemos quantas vezes lançamos)
$k$
número de sucessos
(observado: contamos as águas)
$p$
probabilidade de sucesso
(desconhecido: o que queremos estimar)

Em R: dbinom(k, size = n, prob = p)

⏱️ 4 min. Enfatize a diferença entre os parâmetros conhecidos (n e k, fixados pelos dados) e o desconhecido (p, o que estamos estimando). Pergunte: "Para p = 0.5, qual a probabilidade de observar 6 águas em 9 lançamentos?" Espere a turma apontar dbinom(6, 9, 0.5) = 0.164. Para p = 0.7: dbinom(6, 9, 0.7) = 0.267. O segundo é mais alto — dados mais compatíveis com p = 0.7.

A verossimilhança como função de $p$

Quando olhamos $P(k \mid n, p)$ como função de $p$ com $k$ e $n$ fixos nos dados, obtemos a verossimilhança:

$$\mathcal{L}(p \mid k, n) = \binom{n}{k} p^k (1-p)^{n-k}$$ 
p_vals   <- seq(0, 1, length.out = 100)
lik_vals <- dbinom(6, size = 9, prob = p_vals)

plot(p_vals, lik_vals, type = "l", lwd = 2,
     xlab = "Proporção de água (p)",
     ylab = "Verossimilhança")
O pico da curva está em $p = k/n = 6/9 \approx 0{,}67$ — o valor mais compatível com os dados.
A largura da curva indica incerteza: valores vizinhos ao pico também têm verossimilhança apreciável.
⏱️ 4 min. Execute o código ao vivo. Pergunte antes: "Antes de rodar, qual valor de p vai ter verossimilhança máxima?" A turma deve convergir para p = k/n = 2/3. Confirme. Pergunte: "Por que isso faz sentido?" A resposta: é o valor de p que torna os dados observados mais prováveis. A curva tem largura apreciável porque há incerteza — com apenas 9 dados, não sabemos p com precisão.

Grid Approximation

Priori, verossimilhança e distribuição posterior

⏱️ 20 min. Este é o núcleo conceitual da aula. Reserve o tempo todo. Escreva as três etapas no quadro antes de mostrar o código. O processo deve ficar claro antes de qualquer linha de R.

Três etapas, a mesma lógica da Aula 01

1
Definir o grid e a priori $P(p_i)$

Discretizar $[0,1]$ em $G$ pontos. Atribuir a priori a cada ponto.

2
Calcular a verossimilhança $\mathcal{L}(p_i \mid k, n)$

Para cada $p_i$ do grid: dbinom(k, size = n, prob = p_i)

3
Calcular a posteriori e normalizar

$$P(p_i \mid k, n) = \frac{\mathcal{L}(p_i \mid k, n) \times P(p_i)}{\displaystyle\sum_{j=1}^{G} \mathcal{L}(p_j \mid k, n) \times P(p_j)}$$

⏱️ 3 min. Escreva os três passos no quadro antes de abrir o R. Conecte com a Aula 01: "Antes, fazíamos isso para dois pontos. Agora fazemos para G pontos simultaneamente. A normalização divide pela soma total — garante que os valores somem 1."

Passo 1: O grid e a priori

# Grid de 1000 pontos entre 0 e 1
p_grid <- seq(0, 1, length.out = 1000)

# Priori uniforme: todos os valores igualmente plausíveis
prior <- rep(1, 1000)
Priori uniforme — $P(p_i) = 1$ para todo $p_i$. Expressa ignorância total: antes de ver os dados, não temos preferência por nenhum valor de $p$.
Com 1000 pontos, a aproximação é tão precisa que se comporta como uma distribuição contínua para fins práticos.
⏱️ 3 min. Explique que a priori uniforme é apenas uma das possibilidades. Ela será contrastada na próxima parte. Mostre que rep(1, 1000) gera um vetor de uns — todos os pontos com o mesmo peso antes dos dados.

Passo 2: A verossimilhança no grid

# k = 6 águas em n = 9 lançamentos
k <- 6
n <- 9

# Para cada ponto do grid, calcular dbinom(k, n, p_i)
likelihood <- dbinom(k, size = n, prob = p_grid)
O resultado é um vetor de 1000 valores: a verossimilhança de cada hipótese $p_i$ para os dados observados.
O pico do vetor está em $p = k/n \approx 0{,}67$ — o valor mais compatível com $k = 6$, $n = 9$.
⏱️ 3 min. Mostre que likelihood é um vetor com o mesmo comprimento que p_grid (1000 valores). Cada elemento i é dbinom(6, 9, p_grid[i]). Pergunte: "Qual será o maior valor nesse vetor? Em qual posição?" A resposta: próximo ao índice 670 (p ≈ 0.67).

Passo 3: Posteriori e normalização

# Posteriori não normalizada
post_raw  <- likelihood * prior

# Normalização: dividir pela soma garante que os valores somem 1
posterior <- post_raw / sum(post_raw)

# Visualizar
plot(p_grid, posterior, type = "l", lwd = 2,
     xlab = "Proporção de água (p)",
     ylab = "Probabilidade posterior",
     main = "Posteriori: k=6, n=9, priori uniforme")
abline(v = k / n, lty = 2, col = "red")
Por que normalizar? O produto prior * likelihood gera pesos relativos — não somam 1. A divisão pela soma os transforma em probabilidades válidas. Idêntico ao que fizemos na Aula 01 com dois pesos.
⏱️ 4 min. Execute o código. Mostre que sum(post_raw) é um número pequeno (não 1) e sum(posterior) é exatamente 1. Diga: "Isso é a regra de Bayes aplicada a 1000 hipóteses simultaneamente. O gráfico mostra onde a probabilidade está concentrada depois de ver os dados."

Grid grosso versus grid fino

par(mfrow = c(1, 2))

# Grid grosso: 10 pontos
p_grid_10 <- seq(0, 1, length.out = 10)
lik_10    <- dbinom(k, size = n, prob = p_grid_10)
post_10   <- lik_10 / sum(lik_10)
plot(p_grid_10, post_10, type = "b",
     main = "Grid grosso (10 pontos)", xlab = "p", ylab = "Posteriori")

# Grid fino: 1000 pontos
p_grid_1k <- seq(0, 1, length.out = 1000)
lik_1k    <- dbinom(k, size = n, prob = p_grid_1k)
post_1k   <- lik_1k / sum(lik_1k)
plot(p_grid_1k, post_1k, type = "l",
     main = "Grid fino (1000 pontos)", xlab = "p", ylab = "Posteriori")
Mais pontos no grid $\to$ melhor aproximação da distribuição contínua. Com 1000 pontos, a curva é suave.
⏱️ 3 min. O grid grosso mostra bem a forma discreta (pontos conectados), o grid fino parece uma curva contínua. Diga: "A distribuição real é contínua. O grid é uma aproximação. Quanto maior o grid, melhor a aproximação."

O Papel da Priori

Diferentes crenças iniciais, diferentes distribuições posteriores

⏱️ 10 min. Esta parte é o segundo clímax conceitual. Use ao menos dois contrastes. Peça uma previsão dos alunos antes de mostrar cada gráfico.

Três prioris, três posterioris

p_grid <- seq(0, 1, length.out = 1000)
k <- 6; n <- 9
lik <- dbinom(k, size = n, prob = p_grid)

# 1. Uniforme: todos os valores igualmente plausíveis
prior_unif <- rep(1, 1000)
post_unif  <- lik * prior_unif / sum(lik * prior_unif)

# 2. Escalonada: descarta p < 0.5 (crença de que há mais água)
prior_esc  <- ifelse(p_grid < 0.5, 0, 1)
post_esc   <- lik * prior_esc  / sum(lik * prior_esc)

# 3. Concentrada em p ≈ 0.7 (71% de água na Terra)
prior_beta <- dbeta(p_grid, shape1 = 7, shape2 = 3)
post_beta  <- lik * prior_beta / sum(lik * prior_beta)

par(mfrow = c(1, 3))
plot(p_grid, post_unif, type = "l", lwd = 2, main = "Priori uniforme",     xlab = "p")
plot(p_grid, post_esc,  type = "l", lwd = 2, main = "Priori escalonada",   xlab = "p")
plot(p_grid, post_beta, type = "l", lwd = 2, main = "Priori concentrada",  xlab = "p")
Antes de rodar: Se eliminarmos todos os $p < 0{,}5$, onde vocês esperam que a posteriori fique concentrada?
⏱️ 5 min. Execute o código. Espere a turma prever cada resultado antes de mostrar o gráfico. Para a priori escalonada: a posteriori tem probabilidade zero em p < 0.5, toda a massa fica concentrada acima. Para a priori beta: a posteriori sofre um deslocamento em direção a 0.7. Com 9 dados, a priori ainda influencia.

Quando os dados dominam a priori

# Com n = 9 (poucos dados): priori influencia muito
n_pequeno <- 9;   k_pequeno <- 6

# Com n = 90 (mais dados): priori perde influência
n_grande  <- 90;  k_grande  <- 60   # mesma proporção k/n = 2/3

par(mfrow = c(1, 2))
for (i in 1:2) {
  k_i <- c(k_pequeno, k_grande)[i]
  n_i <- c(n_pequeno, n_grande)[i]
  lik_i  <- dbinom(k_i, size = n_i, prob = p_grid)
  post_i <- lik_i * prior_esc / sum(lik_i * prior_esc)  # priori escalonada
  plot(p_grid, post_i, type = "l", lwd = 2,
       main = paste("n =", n_i, "| k =", k_i), xlab = "p")
}
Com $n = 9$, a priori escalonada desloca substancialmente a posteriori. Com $n = 90$, a verossimilhança domina — a posteriori converge para o mesmo resultado independentemente da priori razoável escolhida.
⏱️ 5 min. Pausa de 30 segundos antes de mostrar o gráfico. Pergunte: "Com 90 lançamentos, a priori ainda importa tanto quanto com 9? Por quê?" Depois de ouvir respostas, mostre os dois gráficos. Esta é a propriedade da consistência bayesiana: com evidência suficiente, pesquisadores com prioris distintas convergem para conclusões semelhantes.

Da Posteriori às Previsões

Resumos pontuais, intervalos de credibilidade e distribuição preditiva

⏱️ 15 min. Esta parte conecta a distribuição posterior com aplicações práticas: resumir e usar para prever. É a ponte para as próximas semanas.

Mais dados, menor incerteza

p_grid <- seq(0, 1, length.out = 1000)
prior  <- rep(1, 1000)

# Mesma proporção k/n = 2/3, tamanhos de amostra crescentes
pares <- list(c(k=2, n=3), c(k=6, n=9), c(k=20, n=30), c(k=60, n=90))

par(mfrow = c(2, 2))
for (par in pares) {
  lik  <- dbinom(par["k"], size = par["n"], prob = p_grid)
  post <- lik * prior / sum(lik * prior)
  plot(p_grid, post, type = "l", lwd = 2,
       main = paste("k =", par["k"], "/ n =", par["n"]),
       xlab = "p", ylab = "Posteriori")
  abline(v = 2/3, lty = 2, col = "red")
}
O pico permanece em $p \approx 2/3$ — a proporção observada. A largura diminui com $n$: mais dados, posteriori mais estreita, menor incerteza.
⏱️ 3 min. Use a analogia da pesquisa de intenção de voto: "Se você entrevistar 3 pessoas e 2 disseram sim, a estimativa é 67%, mas com grande incerteza. Se entrevistar 90 e 60 disseram sim, a estimativa é a mesma, mas a incerteza é muito menor." A posteriori captura exatamente essa diferença.

p_grid <- seq(0, 1, length.out = 1000)
lik    <- dbinom(6, size = 9, prob = p_grid)
post   <- lik / sum(lik)   # priori uniforme implícita

# MAP: ponto de maior probabilidade posterior
map <- p_grid[which.max(post)]
cat("MAP:", round(map, 3), "\n")

# Média posterior
media_post <- sum(p_grid * post)
cat("Média posterior:", round(media_post, 3), "\n")
⏱️ 3 min. Para priori uniforme, o MAP coincide com a estimativa de máxima verossimilhança k/n. A média e mediana diferem levemente porque a distribuição é levemente assimétrica para amostras pequenas. Diga: "Esses três números resumem a distribuição em um único valor. Mas o valor mais rico é a distribuição completa — que usamos para calcular intervalos."

Intervalos de credibilidade

Intervalo de credibilidade de $X\%$: intervalo $[a, b]$ tal que $P(a \leq p \leq b \mid \text{dados}) = X/100$. Interpretação direta: dados os dados observados, a probabilidade de $p$ estar no intervalo é $X\%$. 
# Amostrar 10.000 valores da posteriori
set.seed(42)
amostras <- sample(p_grid, size = 10000, replace = TRUE, prob = post)

# Intervalo de 90%: do percentil 5% ao percentil 95%
IC_90 <- quantile(amostras, probs = c(0.05, 0.95))
cat("IC 90%:", round(IC_90, 3), "\n")

# Probabilidade direta: P(p > 0.5 | dados)
prob_acima <- mean(amostras > 0.5)
cat("P(p > 0.5 | dados):", round(prob_acima, 3), "\n")
O intervalo de confiança frequentista de 95% significa algo diferente: se o experimento fosse repetido muitas vezes, 95% dos intervalos calculados conteriam o valor verdadeiro. No intervalo de credibilidade, a probabilidade refere-se diretamente ao parâmetro.
⏱️ 4 min. Mostre que sample() com prob = post gera amostras proporcionais à probabilidade posterior. Valores de p com alta probabilidade posterior aparecem com mais frequência nas amostras. A função quantile() calcula percentis dessas amostras. Execute e mostre o resultado: IC_90 deve ser aproximadamente [0.44, 0.86] para k=6, n=9.

Distribuição preditiva posterior

Usando a posteriori para fazer previsões sobre novos dados:

# Para cada amostra de p, simular 10 novos lançamentos
pred <- rbinom(10000, size = 10, prob = amostras)

barplot(table(pred) / 10000,
        main = "Distribuição preditiva (10 novos lançamentos)",
        xlab = "Número de 'águas' previstas",
        ylab  = "Probabilidade preditiva",
        col   = "salmon")
rbinom() incorpora a variabilidade do processo aleatório (lançamentos).
amostras incorpora a incerteza sobre $p$ (largura da posteriori).
⏱️ 4 min. Explique a diferença entre dbinom e rbinom: dbinom calcula probabilidades, rbinom gera dados aleatórios. A distribuição preditiva incorpora dois tipos de incerteza: (1) incerteza sobre p (capturada pela posteriori) e (2) variabilidade aleatória do processo binomial (capturada por rbinom). Posterioris mais estreitas geram distribuições preditivas mais concentradas.

Exemplo com Dados Reais

Proporção de adultos acima de 160 cm — dados Howell1

⏱️ 5 min (demonstração rápida ou leitura prévia do aluno). Esta parte ancora o modelo binomial em dados reais. Pode ser apresentada como transição para o laboratório.

Estimando uma proporção com dados reais

Dataset de 544 indivíduos do povo !Kung San (Kalahari). Dados: McElreath, 2020.

Pergunta: qual é a proporção $p$ de adultos com altura superior a 160 cm?

# Carregar os dados
howell  <- read.csv(
  "_bibliography/mcelreath-2019/mcelreath-2019-data-sets/Howell1.csv",
  sep = ";"
)
adultos   <- howell[howell$age >= 18, ]
k_altura  <- sum(adultos$height > 160)
n_altura  <- nrow(adultos)
cat("Adultos:", n_altura, "| Acima de 160 cm:", k_altura, "\n")

# Grid approximation
p_grid <- seq(0, 1, length.out = 1000)
lik    <- dbinom(k_altura, size = n_altura, prob = p_grid)
post   <- lik / sum(lik)

plot(p_grid, post, type = "l", lwd = 2,
     xlab = "p", ylab = "Posteriori",
     main = "Proporção de adultos acima de 160 cm")
Com ~350 adultos, a posteriori é estreita — muito mais informativa do que com 9 lançamentos do globo.
⏱️ 5 min. Execute o código. Compare a largura desta posteriori com a do modelo do globo (n=9). A diferença é dramática — com n≈350, a incerteza sobre p é muito menor. Use como motivação para a regra geral: mais dados = posteriori mais estreita.
