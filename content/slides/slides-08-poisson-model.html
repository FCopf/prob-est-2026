Modelos de Poisson para Dados de Contagem

Boas-vindas. Esta aula parte exatamente do ponto em que o estudo do modelo linear encerrou: a checagem preditiva da posteriori do modelo Normal aplicado ao dataset RIKZ revelou uma limitação fundamental. O modelo simulava valores negativos de riqueza de espécies, o que é impossível para uma variável de contagem. O objetivo hoje é resolver essa incompatibilidade com o GLM de Poisson. Verificar se todos realizaram a leitura prévia. Tempo estimado para esta parte: 5 min.

O que o modelo linear deixou em aberto

Dataset RIKZ: riqueza de espécies bentônicas em função do nível de maré. Dados: Zuur et al., 2009.

O modelo Normal bayesiano foi ajustado. A checagem preditiva da posteriori revelou um problema.

Problema identificado: o modelo Normal simulava valores negativos de riqueza de espécies. Riqueza é uma contagem: um inteiro não negativo. O modelo Normal atribui probabilidade positiva a qualquer valor real.
⏱️ 2 min. Reativar o problema do estudo do modelo linear antes de qualquer coisa nova. Perguntar: "Quem se lembra do que o pp_check() mostrou no modelo Normal para o RIKZ?" Aguardar respostas antes de avançar. O ponto central é que a incompatibilidade não é um acidente de ajuste: ela é estrutural. A Distribuição Normal é contínua e não respeita o zero como mínimo.

Por que o modelo Normal é inadequado para contagens?

Contagens têm
  • Valores inteiros: $\{0, 1, 2, \ldots\}$
  • Limite inferior em zero
  • Variância que cresce com a média
Distribuição Normal tem
  • Valores contínuos em $\mathbb{R}$
  • Probabilidade positiva para $y < 0$
  • Variância $\sigma^2$ constante
A solução: trocar a verossimilhança Normal pela Distribuição de Poisson, que é discreta, não negativa e tem variância proporcional à média.
⏱️ 3 min. Este slide resume a incompatibilidade estrutural. Enfatizar os três pontos da coluna da esquerda como propriedades observáveis dos dados. A coluna da direita mostra por que o modelo Normal viola cada uma delas. A solução não é ajustar melhor o modelo Normal: é trocar a distribuição.

GLMs e a Distribuição de Poisson

Distribuição, função de ligação e o dataset Kline

Esta parte apresenta os dois componentes novos em relação ao modelo linear bayesiano: a Distribuição de Poisson como verossimilhança e a função de ligação logarítmica. O dataset Kline é usado como exemplo concreto para motivar o modelo. Tempo estimado: 15 min.

Modelos Lineares Generalizados: GLMs

Um GLM generaliza o modelo linear em dois sentidos.
1. Distribuição da resposta
Qualquer membro da família exponencial:
Normal, Poisson, Binomial, Gamma, ...
2. Função de ligação
Conecta o preditor linear ao parâmetro da distribuição:
$f(\theta_i) = \beta_0 + \beta_1 x_i$
$$y_i \sim \text{Distribuição}(\theta_i)$$ $$f(\theta_i) = \beta_0 + \beta_1 x_i$$

GLM: sigla em inglês para Generalized Linear Model. Não traduzir.

⏱️ 3 min. Apresentar o GLM como extensão direta do modelo do estudo do modelo linear. No modelo linear estudado anteriormente, a função de ligação era a identidade: f(mu) = mu. No modelo de Poisson, a função de ligação é o logaritmo. A escolha da distribuição e da função de ligação é determinada pela natureza da variável resposta.

A Distribuição de Poisson

$$P(Y = y \mid \lambda) = \frac{e^{-\lambda}\,\lambda^y}{y!}, \quad y \in \{0, 1, 2, \ldots\}$$
Parâmetro $\lambda > 0$
  • Taxa esperada de eventos
  • Único parâmetro da distribuição
Equidispersão
  • $E[Y] = \lambda$
  • $\text{Var}[Y] = \lambda$
  • Média = Variância
Quando $\lambda$ cresce, a distribuição se desloca para valores maiores e se torna mais simétrica. Para $\lambda$ pequeno, a distribuição concentra probabilidade próxima de zero.
⏱️ 3 min. Enfatizar os dois aspectos centrais: a Distribuição de Poisson é discreta e não negativa por construção, e tem apenas um parâmetro. A propriedade E = Var = lambda é chamada de equidispersão. Ela será retomada nos labs quando os estudantes compararem a variância observada dos dados com o modelo.

A função de ligação logarítmica

O preditor linear $\beta_0 + \beta_1 x_i$ pode assumir qualquer valor real. O parâmetro $\lambda_i$ deve ser estritamente positivo.
$$\log(\lambda_i) = \beta_0 + \beta_1 x_i \quad \Longleftrightarrow \quad \lambda_i = e^{\,\beta_0 + \beta_1 x_i}$$
Como $e^z > 0$ para qualquer $z$ real, a taxa esperada $\lambda_i$ é sempre positiva. O logaritmo é a função de ligação que transforma o preditor linear em um parâmetro positivo.
Atenção: a função de ligação transforma o parâmetro $\lambda_i$, não a variável resposta $y_i$. A expressão correta é $\log(\lambda_i) = \ldots$, não $\log(y_i) = \ldots$
⏱️ 3 min. Este é o ponto de maior confusão. Muitos estudantes escrevem log(y_i) ao invés de log(lambda_i). Enfatizar: a transformação é sobre o parâmetro da distribuição, não sobre os dados. log(y_i) excluiria os zeros e trataria dados transformados como contínuos: seria um modelo completamente diferente.

Dataset Kline: ferramentas em ilhas da Oceânia

Dados: McElreath, 2020. 10 sociedades insulares da Oceânia.
Variáveis
  • total_tools: número de ferramentas tecnológicas
  • population: tamanho da população
  • contact: nível de contato entre ilhas
Por que o modelo de Poisson?
  • total_tools é uma contagem inteira
  • Populações variam em ordens de grandeza
  • Relação esperada é multiplicativa
Hipótese: sociedades maiores mantêm mais ferramentas porque têm mais pessoas para inventá-las e transmiti-las.
⏱️ 2 min. O dataset Kline é usado exclusivamente na parte teórica desta aula. O laboratório usa o dataset RIKZ, que os estudantes já conhecem. O contexto de ferramentas em ilhas torna intuitiva a relação entre tamanho populacional e número de ferramentas, e motiva naturalmente o uso do logaritmo da população como covariável.

Log da população como covariável

Escala absoluta de população
Relação não linear entre ferramentas e população bruta. Populações variam de centenas a dezenas de milhares.
Escala log da população
Relação aproximadamente linear entre ferramentas e log(população). Comprime diferenças de ordem de grandeza.
$$\log(\lambda_i) = \beta_0 + \beta_1 \cdot \log(\text{pop}_i)_c$$

O subscrito $c$ indica centralização: $\log(\text{pop}_i)_c = \log(\text{pop}_i) - \overline{\log(\text{pop})}$

⏱️ 2 min. A motivação para usar o logaritmo da população é tanto teórica (relação multiplicativa) quanto empírica (a relação é mais linear na escala log). A centralização do log(pop) facilita a interpretação do intercepto como a taxa esperada de ferramentas para uma ilha com tamanho populacional médio.

(B) O modelo de Poisson estima a variância separadamente. O Normal não.

(D) O modelo de Poisson exige que os dados sejam padronizados antes do ajuste.

Componentes, Interpretação e Distribuições A Priori

Escala log, interpretação multiplicativa

Esta parte constrói a estrutura completa do GLM de Poisson: os três componentes, a interpretação multiplicativa de beta_1 e as distribuições a priori na escala log. A conexão com o modelo linear bayesiano é direta: a lógica dos componentes é idêntica, apenas os detalhes mudam. Tempo estimado: 15 min.

Os três componentes do GLM de Poisson

Modelo de Poisson para o dataset Kline:
$$\text{total\_tools}_i \sim \text{Poisson}(\lambda_i)$$ $$\log(\lambda_i) = \beta_0 + \beta_1 \cdot \log(\text{pop}_i)_c$$ $$\beta_0 \sim \text{Normal}(3,\,0{,}5) \qquad \beta_1 \sim \text{Normal}(0,\,0{,}2)$$
Componente Expressão Papel
Distribuição $\text{total\_tools}_i \sim \text{Poisson}(\lambda_i)$ Incerteza estocástica
Preditor linear $\log(\lambda_i) = \beta_0 + \beta_1 x_i$ Taxa esperada determinística
Distribuições a priori $\beta_0 \sim \ldots$, $\beta_1 \sim \ldots$ Crenças antes dos dados
⏱️ 2 min. A estrutura de três componentes é idêntica à do estudo do modelo linear. As diferenças são: (1) a distribuição muda de Normal para Poisson, e (2) o preditor linear opera sobre log(lambda_i), não diretamente sobre lambda_i. A tabela ajuda a visualizar essas diferenças de forma comparativa com o que foi visto anteriormente.

Interpretação multiplicativa de $\beta_1$

Modelo Normal
$\beta_1$ = mudança aditiva na média por unidade de covariável.
"Cada kg adicional de peso está associado a $\beta_1$ cm adicionais de altura."
Modelo de Poisson
$\beta_1$ = mudança multiplicativa na taxa por unidade de covariável.
"Cada unidade adicional de $x_i$ multiplica a taxa esperada por $e^{\beta_1}$."
$$\frac{\partial \log(\lambda_i)}{\partial x_i} = \beta_1 \quad \Longrightarrow \quad \lambda_i \text{ se multiplica por } e^{\beta_1} \text{ por unidade de } x_i$$
⏱️ 3 min. Este é o ponto de maior dificuldade para estudantes acostumados ao modelo Normal. A mudança de aditivo para multiplicativo é fundamental. Pedir que calculem no papel: se beta_1 = 0,4, qual é o fator multiplicativo? e^0,4 = 1,49. Se beta_1 = -0,4? e^{-0,4} = 0,67. Esses cálculos serão feitos no Lab 2.

Interpretação de $\beta_0$ e $\beta_1$: exemplos

Dataset Kline com $\log(\text{pop})$ centralizado.
Intercepto $\beta_0 = 3$
Taxa esperada para uma ilha com tamanho populacional médio:
$\lambda = e^3 \approx 20$ ferramentas
Coeficiente $\beta_1 = 0{,}4$
Cada unidade adicional de $\log(\text{pop})_c$ multiplica a taxa por:
$e^{0{,}4} \approx 1{,}49$ (aumento de 49%)
Para $\beta_1 < 0$: o fator multiplicativo $e^{\beta_1} < 1$ indica redução da taxa a cada unidade de covariável. Por exemplo, $\beta_1 = -0{,}4 \Rightarrow e^{-0{,}4} \approx 0{,}67$: redução de 33%.
⏱️ 2 min. Os exemplos numéricos tornam concreta a interpretação multiplicativa. Perguntar: "E se beta_1 = 0? O que isso significa?" e^0 = 1: a taxa não muda com a covariável. Este é um caso de independência.

Distribuições a priori na escala log

A escolha das distribuições a priori exige raciocínio na escala log.
Priori para $\beta_0$
Ilhas com população média: ~20 ferramentas.
$\log(20) \approx 3$. Com $\sigma = 0{,}5$, 95% de probabilidade entre $e^2 \approx 7$ e $e^4 \approx 55$ ferramentas. $$\beta_0 \sim \text{Normal}(3,\,0{,}5)$$
Priori para $\beta_1$
$|\beta_1| = 1 \Rightarrow$ fator $e \approx 2{,}7$: muito grande. Com $\sigma = 0{,}2$, 95% de $\beta_1$ entre $-0{,}4$ e $0{,}4$: fatores entre $0{,}67$ e $1{,}49$. $$\beta_1 \sim \text{Normal}(0,\,0{,}2)$$
⏱️ 2 min. O raciocínio de três passos para a priori do intercepto: (1) taxa esperada plausível no valor médio da covariável, (2) log dessa taxa, (3) desvio padrão que cobre uma faixa razoável. Para o coeficiente, o raciocínio é pelo tamanho do efeito multiplicativo.

O perigo das distribuições a priori "vagas" na escala log

$\beta_0 \sim \text{Normal}(0, 10)$ parece "vaga" na escala log. Na escala de contagem, ela é fortemente informativa.
Na escala log
$\beta_0 \sim \text{Normal}(0, 10)$: distribuição aparentemente suave, centrada em zero.
Na escala de contagem
$\lambda = e^{\beta_0}$: 95% da probabilidade inclui taxas entre $e^{-20}$ e $e^{20}$. Inclui milhões de ferramentas e taxas próximas de zero.
A checagem preditiva a priori com pp_check() e sample_prior = "only" é a ferramenta mais direta para diagnosticar distribuições a priori inadequadas.
⏱️ 2 min. Este slide responde à Enquete 2. A transformação exponencial amplifica os extremos da distribuição Normal. Uma distribuição vaga na escala log traduz-se em taxas absurdamente grandes ou próximas de zero na escala original. Mostrar o histograma de exp(Normal(0, 10)) reforça visualmente o argumento.

Enquete 2

Por que a distribuição a priori $\beta_0 \sim \text{Normal}(0, 10)$ pode ser problemática no modelo de Poisson, mesmo parecendo vaga?

(A) Porque essa distribuição é sempre muito estreita na escala de contagem.

(B) Porque $e^{\text{Normal}(0,\,10)}$ produz taxas esperadas com valores absurdamente grandes ou próximas de zero.

(C) Porque a distribuição Normal não pode ser usada como distribuição a priori em GLMs.

(D) Porque o intercepto deve sempre ter distribuição a priori Lognormal no modelo de Poisson.

⏱️ 2 min. Resposta correta: (B). Uma priori aparentemente difusa na escala log é fortemente informativa na escala de contagem. A checagem preditiva a priori revela esse problema antes mesmo de observar os dados. As alternativas (C) e (D) são incorretas: Normal é uma escolha válida para prioris em GLMs, e não há restrição sobre Lognormal.

Checagem A Priori, Ajuste e Diagnóstico

Da especificação das prioris à distribuição a posteriori

Esta parte apresenta o fluxo completo de ajuste do modelo de Poisson: checagem preditiva a priori, ajuste com os dados, diagnóstico de convergência e extração da distribuição a posteriori. O procedimento é estruturalmente idêntico ao do estudo do modelo linear. Tempo estimado: 10 min.

Checagem preditiva a priori: código

library(brms)

# Centralizar log(population)
kline$log_pop_c <- log(kline$population) - mean(log(kline$population))

# Especificar as distribuições a priori
prioris_kline <- c(
  prior(normal(3, 0.5), class = Intercept),
  prior(normal(0, 0.2), class = b)
)

# Ajustar usando apenas as distribuições a priori
fit_kline_prior <- brm(
  formula      = total_tools ~ log_pop_c,
  family       = poisson(link = "log"),
  prior        = prioris_kline,
  data         = kline,
  sample_prior = "only",
  chains = 4, iter = 2000, warmup = 1000, refresh = 0
)

pp_check(fit_kline_prior, ndraws = 100)
⏱️ 2 min. O código é quase idêntico ao do estudo do modelo linear. As únicas mudanças são: family = poisson(link = "log"), ausência de priori para sigma (Poisson não tem sigma separado), e os valores das prioris calibrados na escala log. Chamar atenção para sample_prior = "only": este argumento é o que faz o modelo amostrar apenas das prioris, sem usar os dados.

O que o pp_check() deve mostrar

Resultado adequado
  • Contagens estritamente não negativas
  • Faixa plausível: 10 a 100 ferramentas
  • Sem contagens com ordens de grandeza absurdas
Sinal de problema
  • Contagens na casa de milhões
  • Distribuição completamente incompatível com o contexto
  • Prioris mal calibradas na escala log
A checagem preditiva a priori antecede o ajuste com os dados. É tão importante quanto a verificação da convergência.
⏱️ 2 min. A checagem preditiva a priori é uma etapa de design, não de diagnóstico posterior. Se o pp_check() mostrar contagens com ordens de grandeza absurdas, as prioris devem ser revisadas antes de qualquer ajuste. Esse princípio vale para qualquer GLM bayesiano.

Ajuste completo e diagnóstico

# Ajuste completo (remover sample_prior = "only")
fit_kline <- brm(
  formula = total_tools ~ log_pop_c,
  family  = poisson(link = "log"),
  prior   = prioris_kline,
  data    = kline,
  chains  = 4, iter = 2000, warmup = 1000, refresh = 0
)

# Diagnóstico de convergência
summary(fit_kline)

# Extração da distribuição a posteriori
library(posterior)
amostras_kline <- as_draws_df(fit_kline)
summarise_draws(amostras_kline, mean, sd, ~quantile(.x, c(0.025, 0.975)))
Critérios de convergência: $\hat{R} < 1{,}01$ e Bulk ESS $> 400$ para todos os parâmetros.
⏱️ 2 min. O código de ajuste é idêntico ao do estudo do modelo linear, exceto pela troca de family. Os critérios de convergência são os mesmos. Enfatizar que o diagnóstico sempre antecede a interpretação das estimativas.

Interpretação na escala de contagem

Os valores brutos de $\beta_0$ e $\beta_1$ estão na escala log. A interpretação exige a transformação com exp(). 
# Interpretar na escala de contagem
exp(mean(amostras_kline$b_Intercept))   # ferramentas esperadas no log(pop) médio
exp(mean(amostras_kline$b_log_pop_c))   # fator multiplicativo por unidade de log(pop)_c
$e^{\hat{\beta}_0}$
Número esperado de ferramentas para uma ilha com $\log(\text{pop})_c = 0$ (população média).
$e^{\hat{\beta}_1}$
Fator multiplicativo: a taxa esperada de ferramentas é multiplicada por esse valor a cada unidade de $\log(\text{pop})_c$.
⏱️ 2 min. Reforçar: exp(beta_0) é a taxa esperada no centro da covariável centralizada. exp(beta_1) é o fator multiplicativo. Sem a transformação exp(), os valores têm interpretação na escala log, que é indireta.

Faixa de Credibilidade e Intervalo Preditivo

Duas formas de incerteza no modelo de Poisson

Esta parte apresenta a distinção entre a faixa de credibilidade (incerteza sobre a curva média) e o intervalo preditivo (incerteza sobre contagens individuais). No modelo de Poisson, o intervalo preditivo é heteroscedástico: cresce com lambda_i. Esse contraste com o modelo Normal do estudo do modelo linear é o ponto pedagógico central. Tempo estimado: 5 min de teoria mais quadro de correspondências.

Duas formas de incerteza

Faixa de credibilidade
Calculada com posterior_epred().
Incerteza sobre a taxa esperada $\lambda_i$ (o parâmetro do modelo).
Banda azul nos gráficos.
Intervalo preditivo
Calculado com posterior_predict().
Incerteza sobre uma nova contagem individual $y_*$.
Banda laranja nos gráficos.
No modelo de Poisson: $\text{Var}[y_i] = \lambda_i$. A banda laranja cresce com $\lambda_i$: ilhas com mais ferramentas esperadas têm contagens mais dispersas. Esse comportamento é diferente do modelo Normal do estudo do modelo linear, onde $\sigma^2$ era constante.
⏱️ 3 min. Esta é a distinção central entre epred e predict, retomada do estudo do modelo linear mas com novo detalhe: no modelo de Poisson, a banda preditiva é heteroscedástica por construção. A largura da banda laranja não é constante: ela cresce onde lambda_i é maior. Isso ocorre porque Var = lambda no modelo de Poisson. No modelo Normal, sigma^2 era constante, então a banda laranja era sempre a mesma largura ao longo de toda a faixa de x.

Construindo as duas bandas: código

log_pop_seq  <- seq(min(kline$log_pop_c), max(kline$log_pop_c), length.out = 100)
novos_dados  <- data.frame(log_pop_c = log_pop_seq)

# Faixa de credibilidade da taxa esperada
epred_kline <- posterior_epred(fit_kline, newdata = novos_dados)
resumo_epred <- data.frame(
  log_pop_c = log_pop_seq,
  media = apply(epred_kline, 2, mean),
  q_inf = apply(epred_kline, 2, quantile, 0.055),
  q_sup = apply(epred_kline, 2, quantile, 0.945)
)

# Intervalo preditivo
ppred_kline <- posterior_predict(fit_kline, newdata = novos_dados)
resumo_pred <- data.frame(
  log_pop_c = log_pop_seq,
  q_inf = apply(ppred_kline, 2, quantile, 0.055),
  q_sup = apply(ppred_kline, 2, quantile, 0.945)
)
⏱️ 2 min. O código segue o mesmo padrão do estudo do modelo linear. A diferença é que posterior_predict() produz contagens inteiras simuladas da Distribuição de Poisson, enquanto posterior_epred() produz a taxa esperada contínua. No laboratório, os estudantes usarão o mesmo padrão aplicado ao dataset RIKZ.

Visualização: curva ajustada com as duas bandas

ggplot(kline, aes(x = log_pop_c, y = total_tools)) +

A banda laranja é mais larga para populações maiores (mais ferramentas esperadas, maior $\lambda_i$). A maior parte dos pontos observados deve estar dentro da banda laranja.
⏱️ 2 min. Chamar atenção para a heteroscedasticidade da banda laranja. A curva média é não linear na escala original da população, mas linear na escala log. Essa não-linearidade é uma propriedade do modelo de Poisson: a relação entre lambda e a covariável é exponencial na escala original.

Dúvidas?
